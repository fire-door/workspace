# 🎯 OpenClaw 实战案例库

## 💼 工作场景

### 案例 1: 每日天气推送 🌤️
**场景：** 每天早上7点自动推送天气到飞书

**实现：** 定时任务 + weather 技能

**配置：**
```bash
# 添加定时任务
openclaw cron add --name "daily-weather-hangzhou" \
  --schedule "0 7 * * *" \
  --command "查询杭州今天的天气，包括温度、天气状况、穿衣建议"
```

**效果：**
- ⏰ 每天 7:00 AM 自动执行
- 📱 推送到飞书 DM
- 🌡️ 包含温度、天气、穿衣建议

**扩展：**
- 多城市天气（杭州、宁波、安庆）
- 工作日/周末不同时间
- 特殊天气提醒（下雨、降温）

### 案例 2: 飞书文档自动化 📄
**场景：** 自动整理会议纪要到飞书文档

**实现：** feishu-doc 技能 + 自定义工作流

**工作流：**
1. 接收飞书消息（会议内容）
2. 解析并格式化内容
3. 创建飞书文档
4. 返回文档链接

**示例对话：**
```
用户: 帮我整理以下会议纪要：
【产品评审会】
时间：2026-03-07 10:00
参会：张三、李四、王五
内容：
1. 新功能上线时间确定
2. UI 优化需求
3. 性能优化计划

助手: 已为你创建飞书文档：
📋 2026-03-07 产品评审会纪要
🔗 https://feishu.cn/docx/xxx
```

**技能组合：**
- feishu-doc（文档创建）
- feishu-wiki（知识库管理）

### 案例 5: 健康提醒 💪
**场景：** 定时提醒喝水、休息

**实现：** 定时任务 + 消息推送

**配置：**
```bash
# 喝水提醒（每2小时）
openclaw cron add --name "water-reminder" \
  --schedule "0 9,11,13,15,17,19 * * *" \
  --command "提醒该喝水了，保持健康！"

# 休息提醒（每1.5小时）
openclaw cron add --name "rest-reminder" \
  --schedule "0 10,11:30,13,14:30,16,17:30 * * *" \
  --command "该休息一下了，站起来活动活动！"
```

### 案例 8: 代码审查助手 👨‍💻
**场景：** 自动审查 GitHub Pull Requests

**实现：** agent-browser + GitHub API

**工作流：**
1. 监听 GitHub Webhook
2. 自动打开 PR 页面
3. 分析代码变更
4. 生成审查意见
5. 推送到飞书

**示例：**
```
🔍 代码审查报告

PR #123: 优化数据库查询
变更文件：3个
代码行数：+120, -45

⚠️ 发现问题：
1. src/db/query.ts:45 - 缺少错误处理
2. src/api/user.ts:78 - SQL 注入风险

💡 建议：
- 添加 try-catch 错误处理
- 使用参数化查询
```

